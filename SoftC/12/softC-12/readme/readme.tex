\section{はじめに}

本アプリケーションはユーザが今までに作成したCプログラムファイルの管理を行うためのものである。本アプリケーションを用いることで、作成したCソースファイル、実行ファイル、Makefileなどのプログラム関連のファイルや、図形ファイル、TeXファイルなどレポートに関するファイルまで管理することが可能となる。アプリケーションでユーザが選択できる機能として、以下のものが挙げられる。
\begin{enumerate}
\item 新規ファイルデータの追加
\item 既存ファイルデータの削除
\item 既存ファイルデータの参照
\item 任意のファイル、任意の項目に対しての修正
\item 新規分類項目の追加
\item 既存分類項目の削除
\item ファイル表示の際のソート
\end{enumerate}

また、本アプリケーションが自動で行う機能として、以下のものが挙げられる。
\begin{enumerate}
\item ファイルの読み込み
\item ファイルの保存
\end{enumerate}

本アプリケーションを利用することによって、ユーザはファイルの管理を特殊なコマンドを入力することなく容易に行うことができる。

\section{アプリケーションについて}
\subsection{アプリケーションの起動}
\begin{verbatim}
本アプリケーションを起動するためには、まず端末エミュレータを起動する必要がある。端末エミュレータを起動するには、デスクトップ画面上で右クリックをし、端末を開くを左クリックする。すると端末エミュレータが起動する。次に、起動した端末エミュレータに、「cd softC/softC-12/file_management_application」と入力し、Enterを押す。すると次の行に進むので、「cc -o 」と入力し、Enterを押す。これでアプリケーションが起動する。
正しく起動した場合は「Welcome!」と表示された後、機能一覧が表示される。ユーザはその中の機能を番号を選択することによって選び、アプリケーションを実行する。
\end{verbatim}

\subsection{新規ファイルの登録}

ここでは一つ目の機能である新規ファイルの登録について説明する。新規ファイルの登録は、ユーザが新しいファイルを登録するときに使う機能である。ユーザは説明にしたがって項目を入力することでファイルを登録できる。
\begin{enumerate}
\item タイトル名の入力\\
登録したいファイルのファイル名を入力する。タイトル名が追加機能で必要になるので、実際のファイルと違うファイル名を入力するとそのファイルを検索できなくなる。(例: softC-5-3.c)
\item 副タイトル名の入力\\
登録したいファイルの機能の名前を入力する。この副タイトルでファイルの機能を管理することによって、同じタイトル名で違うプログラムを登録することができる。(例: メインルーチン)
\item 作成日付の入力\\
登録したいファイルの作成日付を入力する。(例: 10/18/2010)
\item パス名の入力
\begin{verbatim}
登録したいファイルのパス名をフルパスで(ルートディレクトリから順に)入力する。パス名が追加機能で必要になるので、実際にファイルが格納されている場所と違うパス名を指定するとそのファイルを検索できなくなる。パス名は必ず「/」で終わること。(例: /home/g031h105/)
\end{verbatim}

\item 分類番号の入力\\
登録したいファイルの分類番号を入力する。この分類番号は予め登録されている分類番号しか入力できない。(例: 3)
\end{enumerate}

\subsection{既存ファイルの削除}
ここでは二つ目の機能である既存ファイルの削除について説明する。既存ファイルの削除は、ユーザが既存のファイルを削除するときに使う機能である。ユーザは説明にしたがって項目を入力することでファイルを削除することができる。ただし、この削除はアプリケーション上のみで行われるものであり、実際のファイルが削除されることはない。
\begin{enumerate}
\item ファイルのタイトル名の入力\\
ユーザは削除するファイルを検索するために、削除したいファイルのタイトル名を入力する。(例: softC-5-3.c)
\item ファイルの副タイトル名の入力\\
ユーザは削除するファイルを検索するために、削除したいファイルの副タイトル名を入力する。削除するファイルの検索はタイトル名と副タイトル名の完全一致で行う。(例: メインルーチン)
\item 削除の確認
削除対象のファイルが見つかったら、本当に削除するかどうかを選択する。ここで選択肢Noを選択すると削除をキャンセルすることができる。Yesを選択すると削除を実行する。
\end{enumerate}

\subsection{任意のファイル、任意の項目への修正}
ここでは三つ目の機能である任意のファイル、任意の項目への修正について説明する。ユーザは既に登録済みの任意のファイル、任意の項目に対して修正をすることができる。
\begin{enumerate}
\item ファイルのタイトル名の入力\\
ユーザは修正するファイルを検索するために、修正したいファイルのタイトル名を入力する。(例: softC-5-3.c)
\item ファイルの副タイトル名の入力\\
ユーザは修正するファイルを検索するために、修正したいファイルの副タイトル名を入力する。修正するファイルの検索はタイトル名と副タイトル名の完全一致で行う。(例: メインルーチン)
\item 修正する項目の選択\\
修正する項目を番号で選択する。本アプリケーションでは一度にひとつの項目しか修正できない。(例: 3)
\item 修正後のデータの入力\\
修正する項目の修正後のデータを入力する。(例: softC)
\end{enumerate}

\subsection{既存のファイルの検索}
ここでは四つ目の機能である既存のファイルの検索について説明する。ユーザは既に登録済みのファイルを任意の項目で検索することができる。また、表示するファイルが複数ある場合はソート方法を選択し、表示させることができる。
\begin{enumerate}
\item 検索する項目の選択\\
タイトル、副タイトル、作成日付、パス名、分類項目の5つの中から1つを選択し、数字で入力する。複数の項目を選択することはできない。
\item 検索キーの入力\\
検索するキーを入力する。
\item ソート方法の選択\\
検索に該当したファイルが複数あった場合は、該当ファイル一覧の表示の際にソートして表示することができる。ユーザはタイトル名昇順副タイトル名昇順、タイトル名昇順副タイトル名降順、タイトル名降順副タイトル名昇順、タイトル名降順副タイトル名降順、作成日付昇順、作成日付降順、パス名昇順、パス名降順、分類番号昇順、分類番号降順の中から選択し、数字を入力する。
\end{enumerate}

\subsection{新規分類項目の追加}
ここでは５つ目の機能である新規分類項目の追加について説明する。ユーザは新たに分類項目を追加し、追加した分類項目をその後のファイル登録などに利用できる。
\begin{enumerate}
\item 分類項目名の入力\\
ユーザは新たに作成したい分類項目名を入力する。分類番号は自動で割り当てられるので入力しない。
\end{enumerate}

\subsection{既存の分類項目の削除}
\begin{enumerate}
\item 分類項目番号の入力\\
ユーザは削除する分類項目を検索するために削除したい分類項目番号を入力する。
\item 分類項目名の入力\\
ユーザは削除する分類項目を検索するために削除したい分類項目名を入力する。削除対象の分類項目の検索は、分類項目番号と分類項目名の完全一致によって行われる。
\item 削除の確認\\
削除対象の分類項目が見つかったら、本当に削除するかどうかを選択する。ここで選択肢Noを選択すると削除をキャンセルすることができる。Yesを選択すると削除を実行する。
\end{enumerate}

\subsection{任意のファイルのコメント抽出}
ここでは拡張機能である任意のファイルのコメント抽出について説明する。この機能は、ユーザが指定した任意のCファイルのコメントのみを表示する機能である。この機能によって、そのファイルのおおまかな内容を把握することができる。
\begin{enumerate}
\item ファイルのタイトル名の入力\\
ユーザはコメントを抽出するファイルを検索するために、コメント抽出したいファイルのタイトル名を入力する。(例: softC-5-3.c)
\item ファイルの副タイトル名の入力\\
ユーザはコメントを抽出するファイルを検索するために、コメント抽出したいファイルの副タイトル名を入力する。コメント抽出するファイルの検索はタイトル名と副タイトル名の完全一致で行う。(例: メインルーチン)
\end{enumerate}

\subsection{アプリケーションの終了と実行}
ここではアプリケーションの終了と実行について説明する。ユーザは本アプリケーションの機能を一度実行する毎にアプリケーションの実行を続けるか、終了するかを選択することができる。入力は選択肢の数字によるものとする。
